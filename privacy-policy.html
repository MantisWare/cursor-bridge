CursorBridge MCP - Privacy Policy

TL;DR: CursorBridge MCP processes all data locally on your machine. We don't collect, store, or transmit any personal data to external servers. Your debugging data stays with you.

1. Information We Collect

We collect NO personal information. CursorBridge MCP is designed to process all data locally on your machine.

Data Processed Locally:

  • Console Logs: Browser console messages, errors, and warnings from your active tabs
  • Network Requests: XHR requests, fetch calls, and network responses from your development work
  • Screenshots: Visual captures of your browser tabs for debugging purposes
  • DOM Elements: Information about selected elements on web pages you're debugging
  • Extension Settings: Your preferences for log limits, screenshot paths, and server connections

2. How We Use Your Information

All data processing happens locally on your machine:

  • Local Storage: Console logs and network data are temporarily stored in your browser's local storage
  • Local Server Communication: Data is sent to your locally-running development server (localhost:3035)
  • AI Analysis: Your debugging data is processed by AI tools you configure locally
  • No External Transmission: No data is sent to external servers or third parties

3. Data Storage and Retention

  • Local Storage Only: All data is stored locally in your browser and on your local development server
  • User Control: You can wipe all stored data at any time using the extension's "Wipe Logs" feature
  • Automatic Cleanup: Logs are automatically cleared when you refresh browser tabs or restart your development server
  • No Persistent Storage: We don't maintain any external databases or cloud storage

4. Third-Party Services

We don't use any third-party services. CursorBridge MCP operates entirely with:

  • Your local browser environment
  • Your locally-installed development server
  • Your locally-configured AI development tools

5. Permissions and Why We Need Them

  • activeTab: To capture debugging information from your current tab
  • debugger: To access browser debugging APIs for console logs and network monitoring
  • storage: To save your extension preferences locally
  • tabs: To track tab states and ensure proper debugging functionality
  • tabCapture: To take screenshots for AI analysis
  • windows: To maintain functionality across multiple browser windows
  • host_permissions: To work with any website you're developing (localhost, staging, production)

6. Your Rights and Controls

  • Complete Control: You control all data processing and can stop it at any time
  • Data Deletion: Use the "Wipe Logs" button to delete all stored data immediately
  • Server Control: You control the local development server and can stop it anytime
  • Extension Removal: Uninstalling the extension removes all local data
  • No Account Required: No registration or account creation needed

7. Security

  • Local Processing: All data stays on your machine
  • No Network Transmission: No data is sent over the internet
  • Open Source: All server code is open source and auditable
  • User Control: You have complete control over all components

8. Children's Privacy

CursorBridge MCP is designed for developers and technical users. We don't knowingly collect information from children under 13, and our extension doesn't collect any personal information from anyone.

9. Changes to This Policy

We may update this privacy policy to reflect changes in our extension. We'll notify users of significant changes through the extension or our GitHub repository. Continued use of the extension after changes constitutes acceptance of the updated policy.
