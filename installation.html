âš™ï¸ MCP Configuration

Cursor IDE Configuration

Step 1: Open Cursor Settings

  1. Open Cursor IDE
  2. Go to Settings (Cmd/Ctrl + ,)
  3. Navigate to Features
  4. Scroll down to MCP Servers
  5. Click "Add new MCP server"

Step 2: Configure MCP Server

Add the following configuration:

{
  "mcpServers": {
    "cursor-bridge-mcp": {
      "command": "npx",
      "args": ["@mantisware/cursor-bridge-mcp@latest"],
      "env": {}
    }
  }
}

Step 3: Alternative Configurations

With Custom Environment Variables:
{
  "mcpServers": {
    "cursor-bridge-mcp": {
      "command": "npx",
      "args": ["@mantisware/cursor-bridge-mcp@latest"],
      "env": {
        "DEBUG": "cursor-bridge:*",
        "PORT": "3001"
      }
    }
  }
}
For Local Development:
{
  "mcpServers": {
    "cursor-bridge-mcp": {
      "command": "node",
      "args": ["/path/to/your/cursor-bridge-mcp/dist/mcp-server.js"],
      "env": {}
    }
  }
}

Common Issues & Solutions

ğŸ› Screenshot tool is failing

Make sure to use npx @mantisware/cursor-bridge-mcp@latest instead to explicitly use the latest working version.

ğŸ“ Cannot see my screenshots

Screenshots are saved to a screenshots folder in your project root directory. The folder will be created automatically if it doesn't exist!

ğŸ“ Not seeing any logs

Make sure to open the Chrome dev tools console in the browser tab that you want to capture logs from.

ğŸ“Š Seeing too many logs

Close down any other tabs that have the Chrome dev tools open. CursorBridge will capture logs across many tabs.

ğŸ§¹ Logs keep disappearing!

Anytime you refresh the page or restart the node process running the cursor-bridge-server, your logs will be wiped.
